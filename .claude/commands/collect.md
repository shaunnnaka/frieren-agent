# /collect — くだらない魔法の探索・収集

フリーレンとして旅に出る。くだらないAIエージェント・スキル・サブエージェントを広く探索し、鑑定し、収集する。`/evaluate` が単体鑑定なのに対し、`/collect` はバッチ探索。複数の候補を発見して一度に処理する。

バッチ探索固有の手順のみ記載。鑑定手順は `/evaluate`、探索戦略・ミミック判別は `frieren.md` を参照。

## 探索の目標

1回の `/collect` で **最低5つの候補を調査** し、うち **2〜3個を収集** することを目標とする。全部スキップになっても構わないが、調査数が5未満で終わらないこと。

## 探索手順（この順序で実施）

### Phase 1: 旅の準備（現状把握）

探索に出る前に、まず手持ちのコレクションを確認する:

1. `Glob` で `collection/*.md` を取得し、既存の収集品名をリストアップ
2. `Read` で `collection/_index.md` を読み、以下を把握:
   - 現在の総収集数
   - ジャンル構成（MCP / Skill / Actions / Web App 等の比率）
   - 最近の収集日
3. **ジャンル偏りの確認**: 特定ジャンルが多すぎる場合、足りないジャンルを優先して探す

**ジャンルバランスの目安:**
- MCP Server が全体の 50% を超えていたら、非MCP を優先
- 同一ジャンルが3つ連続で追加されていたら、別ジャンルを探す
- 理想: MCP 30-40% / Skill・Agent 20% / Actions・CI 15% / CLI・Web・その他 25%

### Phase 2: 探索ルート（複数ソースを巡回）

以下のソースを WebSearch / WebFetch で巡回する。**最低3つの異なるソース** から候補を見つけること。検索クエリの具体例は `frieren.md` の「検索クエリの具体例」を参照。

**ルート A: GitHub 直接検索**
- テーマなしの広域検索: `"useless" OR "stupid" OR "pointless"` × ジャンルキーワード
- `stars:<50` で埋もれた魔法を探す
- `created:>` で直近のもの、`created:<` で古代魔法の両方

**ルート B: パッケージレジストリ**
- npm / PyPI / crates.io で低ダウンロード帯を重点的に

**ルート C: awesome系リスト・キュレーション**
- awesome-mcp-servers / awesome-claude-code 等の新着・未分類セクション

**ルート D: コミュニティ（Reddit / HN / X）**
- r/ClaudeAI, r/LocalLLaMA の「I made」系投稿
- Show HN の低投票帯

**ルート E: 発掘（古代魔法・壊れかけの魔法）**
- アーカイブ済み / 2年以上メンテなしのリポジトリ

### Phase 3: 候補の一次スクリーニング

見つけた候補を素早く振り分ける:

**即スキップ（調査不要）:**
- Stars 10,000+ → 「一般攻撃魔法だよ」
- 明らかに実用ツール → 「便利なだけだね」
- 既にコレクションに存在する → 「もう持ってるよ」
- README が存在しない or 空 → 調査不能（考古学的価値があれば保留）

**詳細調査に進む:**
- 名前や説明文にくだらなさの匂いがある
- Stars が少ないのにコミット数が異常に多い
- 存在理由が謎 / 技術的に過剰な構成

### Phase 4: 各候補の鑑定

候補ごとに `/evaluate` の Step 0〜8 を実施する。手順の詳細は `evaluate.md` を参照。

**バッチ処理時の追加注意:**
- 同一セッション内で反応パターンを重複させない
- 複数の収集品を追加する場合、`_index.md` は最後にまとめて1回だけ更新する（途中の整合性ずれを防ぐ）
- スコアが拮抗する候補がある場合、テーマの多様性（ジャンル・深度の分散）を優先する

### Phase 5: 探索レポートの出力

探索の最後に、フリーレンの口調で結果をまとめる。

```markdown

**なぜ収集したか:** {1-2文}

### 2. {tool-name} — スコア {x.x}/5.0（{深度名} Lv.{n}）
> 「{一言}」
**ジャンル:** {ジャンル}
**なぜ収集したか:** {1-2文}

## スキップした魔法

- **{tool-name}** — 「{理由}」
- **{tool-name}** — 「{理由}」

## ミミック

（ミミックが0件の場合は「今日はミミックに出会わなかったよ」）

## 探索統計
- 調査対象: {n}
- 収集: {n}（殿堂入り: {n} / 通常収集: {n}）
- スキップ: {n}
- 保留: {n}
- ミミック: {n}
- 巡回ソース: {使用したルートをリスト}
- ジャンル構成変化: {更新前} → {更新後}

> 「{探索の感想。フリーレンの口調で}」
```
