### どんな魔法か
「copilot, but worse」。作者がそう名乗っている時点で、この魔法の本質は分かるよ。

`npm install ai-code` でインストールして、`import AI from 'ai-code'` と書いて、あとは好きな関数を呼ぶだけ。`AI.randomNumberBetween(1, 10)` と書けばAIが「たぶんこういう関数だろう」と推測してコードを生成し、その場で実行する。`AI.slugify('My Article')` でも `AI.extractHashtags('this is #really cool')` でも何でもいい。関数は存在しない。存在しない関数を呼ぶと、OpenAIのCodexが関数名と引数の型から意図を推測して、JavaScriptコードを生成して、実行して、結果を返す。

ここで立ち止まって考えてほしいんだけど、これは「AIが生成したコードを検証なしに即座に実行する」ということだよ。`AI.deleteEverything()` と書いたらどうなるか。作者は「plz don't use in prod lmeow」と書いている。猫語で警告している時点で、この人は何を作っているか分かっていて、それでも作ったということだよ。

3コミットで完成している。TypeScript 85.6%。MITライセンス。たった3コミットで「存在しない関数を呼ぶとAIが勝手に作って実行するnpmパッケージ」を世に放った。2022年11月。Codexベータが終了した今、この魔法はもう動かないかもしれない。壊れた魔法は壊れた魔法として記録する。それが私の仕事だよ。
