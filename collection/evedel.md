## evedel

> フリーレンの感想：「コードを書かない」（静かに）「コードの中に、指示だけを書く。オーバーレイで追跡して、タグで分類して、LLMに投げる。コードそのものは、人間が書かなくていい」（少し考える）「……それってプログラマーなの。それとも魔法使いなの」

**くだらなさの深度:** 哲学的（Lv.4）— 「コードを書く」行為そのものを疑問に付す。プログラミングの定義を揺さぶっている

### どんな魔法か
Emacsの中で、コードを書く代わりに「指示」を書くパッケージだよ。ソースコード内に指示アノテーションを配置して、それをLLMに投げて、コードを生成させる。指示はEmacsのオーバーレイとして管理されるから、ソースコード自体は汚れない。

普通のAIコーディングツールとの違いは、発想の転換にあるんだよ。普通はコードを書いて、わからないところをAIに聞く。evedelは逆。指示を書いて、コード全体をAIに任せる。タグベースの参照カテゴリ分類があって、複雑なクエリ式（and、or、not）で指示を管理できる。指示は永続化されて、保存・読み込み・自動パッチが効く。バッファを横断して指示をナビゲートできる。LLMに送る前にディレクティブのプレビューまでできる。

98コミット。Emacs Lisp 100%。GPL-3.0。Emacs 29.1以上とgptelパッケージが必要。gptelがサポートするLLM——OpenAI、Claude、ローカルモデル——がそのまま使える。v0.4.20まで更新されている。作者はこの「指示を書くプログラミング」を、少なくとも98回改良し続けている。コードを書かないためのコードを、98回書き直しているんだよ。

### インストール方法
```elisp
;; straight.el
(use-package evedel
  :straight (:host github :repo "daedsidog/evedel"))

;; gptel が事前に設定されている必要がある
;; M-x evedel-create-instruction で指示を作成
```
